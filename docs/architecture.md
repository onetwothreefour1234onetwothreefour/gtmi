## 1. Overview

The Global Talent Mobility Index (GTMI) is a composite indicator platform that benchmarks talent-based premium mobility programs across 30 countries. It combines a 30% Country Mobility Environment (CME) score derived from IMD's published Appeal factor with a 70% Program Architecture Quality (PAQ) score built from 48 indicators across 5 pillars and 15 sub-factors, all sourced exclusively from government documents. Every published value is traceable to an exact sentence in a government source, with scrape timestamp, content hash, character offsets, and extraction model recorded. The index re-scrapes all Tier 1 sources weekly and surfaces policy changes as they occur.

## 3. Data Flow: 7-Stage Extraction Pipeline

The extraction pipeline is defined as TypeScript interfaces in `packages/extraction/src/types/pipeline.ts`. All 7 stage implementations are committed to `packages/extraction/src/stages/`. The Trigger.dev job `extract-single-program` runs the full pipeline end-to-end.

```mermaid
flowchart LR
    job["Trigger.dev\nextract-single-program"]

    subgraph pkg["packages/extraction"]
        s0["Stage 0: Discover\nDiscoverStage\nPerplexity API (sonar)\ndiscovers up to 10 URLs"]
        s1["Stage 1: Scrape\nScrapeStage\nPython/Playwright service\nSHA-256 hash, HTTP status"]
        s2["Stage 2: Extract\nExtractStage\nClaude extracts value,\nconfidence, source sentence\nMulti-URL merge"]
        s2_5["Stage 6.5: Derive (ADR-016)\nDeriveStage\nPure arithmetic, no LLM\nA.1.2 and D.2.2 computed\nfrom extracted inputs +\nstatic lookup tables"]
        s3["Stage 3: Validate\nValidateStage\nSeparate Claude call checks\nsource-sentence alignment"]
        s4["Stage 4: Cross-check\nCrossCheckStage\nCompare against Tier 2 source\nLog disagreements"]
        s5["Stage 5: Human Review\nHumanReviewStage\nQueue if confidence below 0.85\nor cross-check disagrees"]
        s6["Stage 6: Publish\nPublishStage\nWrite field_values row\nwith full provenance chain"]
    end

    job --> s0

    s1 --> s2
    s2 --> s2_5
    s2_5 --> s3
    s3 --> s4
```

Stage 0 uses the **Perplexity API** (`sonar` model, `PERPLEXITY_API_KEY`). Stage 1 uses the **Python/Playwright scraper service** (`SCRAPER_URL`). Stages 2–4 and bulk summaries use `claude-sonnet-4-6`. Four Claude model constants are exported from `packages/extraction/src/clients/anthropic.ts`: `MODEL_EXTRACTION` (Stage 2), `MODEL_VALIDATION` (Stage 3), `MODEL_CROSSCHECK` (Stage 4), and `MODEL_SUMMARY`. `MODEL_DISCOVERY` is also defined but unused by `discover.ts`. Each constant is independently updatable.

**Wave configuration:** `scripts/wave-config.ts` exports `ACTIVE_FIELD_CODES = WAVE_1_FIELD_CODES ∪ (WAVE_2_ENABLED ? WAVE_2_FIELD_CODES : [])`. `WAVE_2_ENABLED = true` (Phase-2-close-out default) → all 48 methodology fields. `WAVE_2_ENABLED = false` reverts to the 27-field Wave 1 scope. All consumers — `canary-run.ts`, `extract-single-program.ts`, `run-paq-score.ts`, `diag-empty-fields.ts` — import `ACTIVE_FIELD_CODES` so a single flag flip changes scope everywhere. Note: Trigger.dev picks up `ACTIVE_FIELD_CODES` at runtime, so production scope changes the moment Trigger.dev redeploys. Staged rollouts to production must flip the flag to `false` before deploy.

**Field-aware content windowing:** `packages/extraction/src/utils/window.ts` exposes `selectContentWindow(content, fields, maxChars)`. The batch extraction path scores 2K-char chunks (200-char overlap) by per-field keyword match against `field_definitions.label`, then greedily fills a 30K budget while preserving a 1500-char baseline prefix and 800-char baseline suffix. Replaces the previous `content.slice(0, 30000)` truncation. Cache key in `extract.ts` includes a `WINDOW_VERSION` constant so windowing changes invalidate stale cache rows cleanly. When no field labels are passed (single-field legacy path), windowing falls back to head-slice — guaranteed never worse than pre-windowing behaviour. The redundant 30K cap on `scrape.ts` was removed.

**Batch extraction + caching:** `executeBatch` extracts all fields for a single scrape in one LLM call (8K max-tokens, JSON array response). `executeAllFields` iterates batches across URLs and merges by highest confidence per field, with a 30s inter-batch delay and an early-exit once every field reaches confidence ≥ 0.9. The `extraction_cache` table (migration `00004`) memoizes results keyed by `sha256(contentHash + fieldKey + promptHash + WINDOW_VERSION)`; cache hits skip the LLM entirely. The `scrape_cache` table memoizes scrape responses for 24h.

**Currency preservation:** `utils/currency.ts` (`detectCurrency()`) recognises 19 ISO 4217 codes and common symbols. `publish.ts` strips the prefix before normalization and writes the code into `provenance.valueCurrency` — no schema change required. `scripts/backfill-monetary-normalization.ts` re-normalizes pending-review rows that were null because the prefix was not stripped pre-fix.

**Two-phase discovery:** Before Stage 0 runs, `scripts/country-sources.ts` country-level sources are loaded and scraped once per country (Phase 1). Stage 0 then discovers program-specific URLs (Phase 2). Per-field in Stage 2, country-level scrapes supplement program-level scrapes. Stage 4 cross-check selects the Tier 2 URL by keyword match against the field label, falling back to global sources from `scripts/country-sources.ts` if no program-level Tier 2 URL matches.

**Self-improving discovery (Phase 3.6 / ADR-015):** After `verifyUrls()` succeeds (cache-hit and cache-miss paths), every surviving discovered URL is upserted into `sources` via `discover.writeToSourcesTable()` keyed on `UNIQUE(program_id, url)`. On conflict, `last_seen_at` bumps and `discovered_by` is set to `'stage-0-perplexity'`; tier is set to `MIN(existing, fresh)` so the registry never downgrades. Before Stage 1 scrapes, `mergeDiscoveredUrls` (in `packages/extraction/src/utils/url-merge.ts`) combines the fresh Stage 0 output with the registry rows from `loadProgramSourcesAsDiscovered(programId)` — filtered to `tier IN (1, 2)`, `last_seen_at > NOW() - 90 days`, and `programs.status='active'`. The merge dedupes by normalised URL (lowercased scheme/host, stripped trailing slash, stripped tracking params), orders Tier 1 → Tier 2 → Tier 3 with quotas (7/4/1), and caps at 12. **From the second run onward, every URL that any prior run successfully verified is guaranteed to be in the scrape set, regardless of Perplexity's current output.** Both `scripts/canary-run.ts` and `jobs/src/jobs/extract-single-program.ts` invoke the merge step at the same logical insertion point via the shared utility.

**V-Dem direct-API for E.3.1 (Phase 3.6 / Fix A):** `fetchVdemRuleOfLawScore(iso3)` fetches the World Bank WGI Rule of Law indicator (`RL.EST`) — methodology v1 line 309 explicitly accepts "V-Dem / World Bank WGI". Pre-fetched in Phase 1 alongside `fetchWgiScore`. E.3.1 published with `extractionModel: 'v-dem-api-direct'`, auto-approved at confidence 1.0. Gated on `PHASE3_VDEM_ENABLED` (default true; set to literal `'false'` to disable). When disabled or fetch returns null, E.3.1 falls through to the LLM extraction batch.

**Stage 6.5 — Derive (Phase 3.6 / ADR-016):** A pure-arithmetic step between Stage 2 (Extract) and Stage 3 (Validate) computes A.1.2 (salary as % of local median wage) and D.2.2 (total years from initial visa entry to citizenship eligibility) from already-extracted inputs plus three static lookup tables: `scripts/country-median-wage.ts` (OECD AAW for OECD members, ILO mean-earnings fallback for non-OECD), `scripts/fx-rates.ts` (annual-average LCU-per-USD), and `scripts/country-citizenship-residence.ts` (years-as-PR before naturalisation; null for no-pathway countries). **No LLM is invoked.** `extractionConfidence` is hard-coded to 0.6 — every derived row routes to /review (auto-approve threshold is 0.85). `publish.executeDerived` writes derived rows to `field_values` with `status='pending_review'`, preserving the full provenance including a `derivedInputs` JSONB extension that lists each input field key, value, and source URL for analyst audit. `<ProvenanceTrigger>` renders a "Computed from:" block when `derivedInputs` is present.

**Stage 6.5 extensions (Phase 3.6.1 / FIX 6 + Phase 3.6.2 / ITEM 2):** D.2.3 (dual-citizenship policy), B.2.4 (mandatory non-government costs), D.1.3 (PR-accrual physical presence), and D.1.4 (PR retention rules) are computed via the same Stage 6.5 derive pattern, reading from country-level static lookup tables (`country-citizenship-policy.ts`, `country-non-gov-costs.ts`, `country-pr-presence.ts` — all under `packages/extraction/src/data/`). Each derive function is country-deterministic (no LLM input), confidence 0.7 (`derived-knowledge`), and routes to /review. The lookup tables carry an analyst-reviewable header note flagging that values must be re-checked before each new country canary.

**Maintenance mode (Phase 3.6.2 / ITEM 3 — `PHASE3_TARGETED_RERUN`):** When the env flag is `'true'`, the canary enters precision mode: Stage 0 receives a "PRECISION BRIEF" listing the still-missing indicator labels and is given the FULL registry as an exclusion list (the default 20-URL slice cap is lifted). LLM extraction is then filtered to the missing-field set. Used after a program has reached ≥42/48 to spend tokens on gap closure rather than another full canary. Default `false`. The derive stage runs regardless — gap closure via Stage 6.5 is not flag-dependent.

**Dynamic discovery cap + provenance pre-loading (Phase 3.6.2 / ITEMs 4+5):** `dynamicUrlCap(populatedFieldCount)` returns 20 / 15 / 12 by coverage band so under-covered programs get breadth and well-covered programs get precision; `dynamicTierQuotas(cap)` scales the 60/30/10 Tier ratio proportionally. `loadProvenUrlsForMissingFields(programId, countryIso, missingKeys)` pre-loads URLs that produced approved rows for the same field key in OTHER programs in the SAME country — the SQL filter on `programs.country_iso` is the cross-country contamination guard. The merge layer accepts these as a third origin (registry → proven → fresh; fresh wins on URL conflict).

**Weekly maintenance scrape (Phase 3.6.2 / ITEM 6 — paused scaffold):** `jobs/src/jobs/weekly-maintenance-scrape.ts` is registered as a Trigger.dev v3 task but has no `schedules.task` block, so it never auto-fires; Phase 5 activates it with cron `0 3 * * 1` to walk the new `field_url_index` view (migration 00012) and emit `policy_changes.severity = 'url_broken'` markers for 404/410 source URLs.

**Scrape thin-content guard + Jina escalation (Phase 3.6 / Fix C):** `MIN_VISIBLE_TEXT_LENGTH` is 1500 (raised from 300 — the AUS Medicare URL returned 484-char redirect-stub content and silently passed the old guard). On `short_content` rejection, `scrape.ts` retries once with `force_layer: 'jina'` in the POST body to short-circuit past Playwright; the Python scraper service honours the parameter. If Jina also returns thin content, the field is logged `SCRAPE_THIN_CONTENT` and treated as ABSENT (no silent zero-fill).

## 5. Methodology Scoring Model

See [docs/METHODOLOGY.md](METHODOLOGY.md) for the authoritative breakdown of sub-factor weights, indicator weights, normalization choices, and rubrics. The diagram below reflects the top-level composite structure as encoded in `packages/db/src/seed/methodology-v1.ts`.

```mermaid
graph TD
    composite["GTMI Composite Score\n0-100"]

    cme["CME: Country Mobility Environment\n30% of composite\nIMD World Talent Ranking Appeal factor\nRe-normalized across the 30-country cohort"]
    paq["PAQ: Program Architecture Quality\n70% of composite"]

    composite --> cme
    composite --> paq

    paq --> A["Pillar A: Access\n28% of PAQ\n3 sub-factors / 9 indicators\nEligibility breadth and selectivity"]
    paq --> B["Pillar B: Process\n15% of PAQ\n3 sub-factors / 10 indicators\nApplication friction, cost, speed"]
    paq --> C["Pillar C: Rights\n20% of PAQ\n3 sub-factors / 10 indicators\nLabor, family, and public service rights"]
    paq --> D["Pillar D: Pathway\n22% of PAQ\n3 sub-factors / 11 indicators\nPR and citizenship progression, tax treatment"]
    paq --> E["Pillar E: Stability\n15% of PAQ\n3 sub-factors / 8 indicators\nPolicy volatility and institutional reliability"]
```

All weights sum to 1.0 at every level. The methodology version is stored in the `methodology_versions` table and carried on every `scores` and `field_values` row. Scores are not recomputed retroactively when the methodology changes; each scoring run carries its version ID.

## 6. Provenance and Data Integrity

### Source tiers

Indicators in Pillars A through D, and in E.1 and E.2, must be populated exclusively from Tier 1 sources. Pillar E.3 uses external indices by design and discloses this on the methodology page.

| Tier   | Sources                                                                             | Use                                                                 |
| ------ | ----------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| Tier 1 | Immigration authority, tax authority, ministry, official gazette, statistics bureau | Primary extraction for all PAQ indicators (Pillars A-D and E.1-E.2) |
| Tier 2 | Law firm commentary (Fragomen, KPMG, Envoy, Baker McKenzie)                         | Cross-check and program narrative context only                      |
| Tier 3 | IMI Daily, Henley newsroom, Expatica, Nomad Gate, general news                      | Policy change early-warning signals only                            |

### Fail-loud posture

The seed script (`packages/db/src/seed/index.ts`) throws immediately on any malformed row. There are no silent skips, no `skip_empty_lines` options, and no default values substituted for missing required fields. Clean CSVs are enforced at the source by `scripts/convert-assets.ts`, which asserts exact row counts after stripping trailing blank lines before writing.

Missing indicator data is not imputed. An absent indicator is excluded from its sub-factor calculation, and a square-root penalty is applied at the sub-factor level. Programs below 70% data coverage on any pillar are flagged "insufficient disclosure" and withheld from the public ranking.
